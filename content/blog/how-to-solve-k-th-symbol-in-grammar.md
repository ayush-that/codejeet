# How to Solve K-th Symbol in Grammar

This problem asks us to find the k-th symbol (1-indexed) in the n-th row of a special binary sequence generated by a simple replacement rule: start with "0", then replace each "0" with "01" and each "1" with "10". While the problem seems straightforward, the exponential growth of the sequence makes brute force generation impossible for larger inputs. The key insight is recognizing the recursive structure and leveraging bit manipulation to solve it efficiently.

## Visual Walkthrough

Let's trace through the generation process for n = 4 to build intuition:

**Row 1:** `0`

**Row 2:** Replace each character in row 1:

- `0` → `01`
  Row 2: `01`

**Row 3:** Replace each character in row 2:

- `0` → `01`
- `1` → `10`
  Row 3: `0110`

**Row 4:** Replace each character in row 3:

- `0` → `01`
- `1` → `10`
- `1` → `10`
- `0` → `01`
  Row 4: `01101001`

Now, if we're asked for k = 5 in row 4 (n = 4, k = 5), we can see the answer is `1`. But notice something interesting: the first half of row 4 (`0110`) is exactly row 3, and the second half (`1001`) is the complement (bitwise NOT) of row 3. This recursive structure is the key to solving this efficiently.

## Brute Force Approach

The most straightforward approach is to actually generate all rows up to n and then return the k-th character. We could implement this by:

1. Starting with row 1 = "0"
2. For each subsequent row i from 2 to n:
   - Create an empty string for the new row
   - For each character in the previous row:
     - If it's '0', append "01"
     - If it's '1', append "10"
3. Return the (k-1)-th character of the final row

The problem with this approach is space and time complexity. Each row doubles in length from the previous one, so row n has 2^(n-1) characters. For n = 30, that's over 500 million characters! This would require gigabytes of memory and take far too long to compute.

```python
# Time: O(2^n) | Space: O(2^n) - IMPRACTICALLY LARGE!
def kthGrammar_brute(n, k):
    # Start with the first row
    row = "0"

    # Generate each subsequent row
    for i in range(2, n + 1):
        new_row = ""
        for char in row:
            if char == '0':
                new_row += "01"
            else:
                new_row += "10"
        row = new_row

    # Return the k-th character (1-indexed, so subtract 1)
    return int(row[k - 1])
```

## Optimized Approach

The key insight comes from observing the recursive structure:

1. **Observation 1**: Row n has exactly 2^(n-1) characters.
2. **Observation 2**: The first half of row n is identical to row (n-1).
3. **Observation 3**: The second half of row n is the complement (bitwise NOT) of row (n-1).

This gives us a recursive relationship:

- If k is in the first half of row n (k ≤ 2^(n-2)), then the answer is the same as kthGrammar(n-1, k)
- If k is in the second half of row n (k > 2^(n-2)), then the answer is the complement of kthGrammar(n-1, k - 2^(n-2))

We can simplify this further by noticing that when k is in the second half, k - 2^(n-2) gives us the position in the previous row, but we need to take the complement of that value.

**Even more elegant approach**: Notice the binary tree structure. Each parent has two children:

- 0 → 01 (left child 0, right child 1)
- 1 → 10 (left child 1, right child 0)

So:

- If k is odd (left child), it matches its parent
- If k is even (right child), it's the complement of its parent

And the parent position in the previous row is ceil(k/2) = (k+1)//2.

This gives us a beautiful recursive formula:

- Base case: n = 1, return 0
- If k is odd: return kthGrammar(n-1, (k+1)//2)
- If k is even: return 1 - kthGrammar(n-1, (k+1)//2)

We can implement this recursively or iteratively. The iterative approach is more efficient as it avoids recursion overhead.

## Optimal Solution

Here's the optimal solution using the recursive relationship with iterative implementation:

<div class="code-group">

```python
# Time: O(n) | Space: O(1)
def kthGrammar(n, k):
    """
    Returns the k-th symbol in the n-th row of the grammar.

    The key insight is recognizing the recursive structure:
    - If k is odd, it matches its parent
    - If k is even, it's the complement of its parent
    The parent is at position ceil(k/2) in the previous row.
    """
    # Start with the value at position 1 in row 1
    current = 0

    # Work backwards from row n to row 1
    # At each step, we find the parent in the previous row
    for row in range(n, 1, -1):
        # Calculate the length of the previous row
        # Row (row-1) has 2^(row-2) elements
        # We can determine if k is in first or second half
        # by comparing with half the length of current row

        # Half length of current row = 2^(row-2)
        half = 1 << (row - 2)  # Equivalent to 2^(row-2)

        if k > half:
            # k is in the second half, so it comes from complement of parent
            # Adjust k to parent position and flip the value
            k -= half
            current = 1 - current

        # If k is in first half, value stays the same
        # and k becomes the parent position (which is k itself)

    return current

# Alternative recursive solution (cleaner but uses O(n) stack space)
def kthGrammarRecursive(n, k):
    # Base case: first row always has 0
    if n == 1:
        return 0

    # Find the parent position in previous row
    parent_pos = (k + 1) // 2

    # Get the parent value
    parent = kthGrammarRecursive(n - 1, parent_pos)

    # Determine if k is left child (odd) or right child (even)
    if k % 2 == 1:
        # Left child: same as parent
        return parent
    else:
        # Right child: complement of parent
        return 1 - parent
```

## Complexity Analysis

**Time Complexity: O(n)**

- We iterate from row n down to row 2, performing constant work at each step
- This gives us exactly n-1 iterations, so O(n) time

**Space Complexity: O(1)** for the iterative solution

- We only use a few integer variables (current, row, half)
- The recursive solution would use O(n) space for the call stack

**Why this is optimal**: Any solution must examine at least n bits of information because the answer depends on the binary representation of k-1 (as we'll see in the bonus insight below).

**Bonus Insight**: There's an even more clever solution that runs in O(log k) time by noticing that the answer is simply the parity of the number of 1's in the binary representation of k-1. This comes from observing that flipping happens when we traverse to a right child (even k). The number of flips equals the number of 1's in the binary representation of k-1.

## Common Mistakes

1. **Off-by-one errors with k indexing**: Remember that k is 1-indexed, not 0-indexed. When calculating half lengths or parent positions, it's easy to mix these up. Always test with small cases like n=1, k=1 (should return 0).

2. **Integer overflow with 2^(n-1)**: For n=30, 2^(29) is about 500 million, which fits in 32-bit integers, but for larger n you might need 64-bit integers. However, the problem constraints keep n ≤ 30, so this isn't an issue here.

3. **Misunderstanding the recursive relationship**: Some candidates think the complement should be applied when k is in the first half instead of the second half. Remember: first half = same as parent, second half = complement of parent.

4. **Using recursion without memoization for the naive approach**: Even with recursion, if you're not using the insight about halves and complements, you'll still have exponential complexity. The recursive solution shown above works because it reduces the problem size by half each time.

## Key Takeaways

1. **Look for recursive structure in generation problems**: When a sequence is generated by rules, there's often a recursive relationship that lets you compute elements directly without generating the entire sequence.

2. **Binary tree thinking can simplify complex recursions**: Mapping the problem to a binary tree (parent-child relationships) often reveals simpler recurrence relations.

3. **Work backwards from the answer**: Instead of building up from row 1 to row n, we worked backwards from position k in row n to find its "ancestor" in row 1. This reverse thinking is powerful for many problems.
