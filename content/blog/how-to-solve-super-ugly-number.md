# How to Solve Super Ugly Number

The Super Ugly Number problem asks us to find the nth positive integer whose prime factors are limited to a given list of primes. This is a generalization of the classic "Ugly Number II" problem (which uses primes [2,3,5]). The challenge lies in efficiently generating these numbers in order without checking every integer, which would be far too slow for large n.

## Visual Walkthrough

Let's trace through a concrete example: `primes = [2,7,13,19]` and `n = 6`.

We need to generate the first 6 super ugly numbers in order. Let's build them step by step:

1. **Start with the first super ugly number**: `1` (by definition)
2. **Initialize pointers**: We'll maintain a pointer for each prime factor, all starting at index 0 (pointing to the first super ugly number, 1)
3. **Generate candidates**:
   - For prime 2: `2 × 1 = 2`
   - For prime 7: `7 × 1 = 7`
   - For prime 13: `13 × 1 = 13`
   - For prime 19: `19 × 1 = 19`
   - The smallest candidate is 2, so that's our second super ugly number
4. **Update pointer**: Only the pointer for prime 2 moves forward (now pointing to super ugly number 2)
5. **Next iteration**:
   - Candidates: `2 × 2 = 4`, `7 × 1 = 7`, `13 × 1 = 13`, `19 × 1 = 19`
   - Smallest is 4 (third super ugly number)
   - Update pointer for prime 2 (now pointing to super ugly number 4)
6. **Continue**:
   - Candidates: `2 × 4 = 8`, `7 × 1 = 7`, `13 × 1 = 13`, `19 × 1 = 19`
   - Smallest is 7 (fourth super ugly number)
   - Update pointer for prime 7 (now pointing to super ugly number 2)
7. **Next**:
   - Candidates: `2 × 4 = 8`, `7 × 2 = 14`, `13 × 1 = 13`, `19 × 1 = 19`
   - Smallest is 8 (fifth super ugly number)
   - Update pointer for prime 2 (now pointing to super ugly number 7)
8. **Next**:
   - Candidates: `2 × 7 = 14`, `7 × 2 = 14`, `13 × 1 = 13`, `19 × 1 = 19`
   - Smallest is 13 (sixth super ugly number)
   - Update pointer for prime 13 (now pointing to super ugly number 2)

So the 6th super ugly number is 13. The sequence is: [1, 2, 4, 7, 8, 13].

## Brute Force Approach

A naive approach would be to check every positive integer in order, factorize it, and see if all its prime factors are in the given list. For each number k, we would:

1. Try to divide k by each prime in the list as many times as possible
2. If we're left with 1, all factors were in the list → it's a super ugly number
3. Count until we find the nth such number

This approach has several problems:

- Factorization is expensive (O(√k) per number)
- We might check many numbers that aren't super ugly before finding the nth one
- For n=1000 and primes=[2], the 1000th super ugly number is 2^999, which is astronomically large

Even a slightly better brute force would generate all multiples of the primes, but we'd need to sort them and remove duplicates, which becomes inefficient for large n due to the exponential growth of the search space.

## Optimized Approach

The key insight is that every super ugly number (after 1) is formed by multiplying a previous super ugly number by one of the primes. This suggests a dynamic programming approach similar to merging k sorted lists.

Think of it this way: For each prime p, we maintain a "stream" of numbers generated by multiplying p with all previously found super ugly numbers. These streams are:

- Stream 1: 1×p, 2×p, 4×p, 7×p, 8×p, 13×p, ...
- Stream 2: 1×q, 2×q, 4×q, 7×q, 8×q, 13×q, ...
- etc.

All these streams are already sorted (since the super ugly numbers are sorted). Our task is to merge these sorted streams to get the next super ugly number at each step.

We can maintain:

1. `dp` array: stores the super ugly numbers we've found so far
2. `pointers` array: for each prime, stores the index of the super ugly number it should multiply with next
3. `candidates` array: the next value from each stream (prime × dp[pointer])

At each step:

- Find the minimum value in `candidates`
- Add it to `dp`
- For all primes that produced this minimum value, advance their pointers and update their candidates
- Continue until we have n numbers

This is essentially a k-way merge where k = number of primes.

## Optimal Solution

Here's the complete implementation using the k-way merge approach:

```python
# Time: O(n * k) where k = len(primes) | Space: O(n + k)
def nthSuperUglyNumber(n: int, primes: list[int]) -> int:
    """
    Returns the nth super ugly number.

    Args:
        n: The position of the super ugly number to find (1-indexed)
        primes: List of prime factors allowed

    Returns:
        The nth super ugly number
    """
    # dp[i] stores the (i+1)th super ugly number (0-indexed)
    dp = [0] * n
    dp[0] = 1  # First super ugly number is always 1

    # pointers[i] stores the index in dp that primes[i] should multiply with next
    pointers = [0] * len(primes)

    # candidates[i] stores the next candidate value from the stream for primes[i]
    candidates = [0] * len(primes)

    # Initialize candidates: each prime multiplied by the first super ugly number (1)
    for i in range(len(primes)):
        candidates[i] = primes[i] * dp[0]

    # Generate the remaining n-1 super ugly numbers
    for i in range(1, n):
        # Find the minimum candidate value
        min_val = min(candidates)
        dp[i] = min_val

        # Update pointers and candidates for all primes that produced min_val
        for j in range(len(primes)):
            if candidates[j] == min_val:
                # Move pointer forward for this prime
                pointers[j] += 1
                # Update candidate: prime × next super ugly number
                candidates[j] = primes[j] * dp[pointers[j]]

    return dp[n-1]
```

function nthSuperUglyNumber(n, primes) {
  // dp[i] stores the (i+1)th super ugly number (0-indexed)
  const dp = new Array(n);
  dp[0] = 1; // First super ugly number is always 1

  // pointers[i] stores the index in dp that primes[i] should multiply with next
  const pointers = new Array(primes.length).fill(0);

  // candidates[i] stores the next candidate value from the stream for primes[i]
  const candidates = new Array(primes.length);

  // Initialize candidates: each prime multiplied by the first super ugly number (1)
  for (let i = 0; i < primes.length; i++) {
    candidates[i] = primes[i] * dp[0];
  }

  // Generate the remaining n-1 super ugly numbers
  for (let i = 1; i < n; i++) {
    // Find the minimum candidate value
    let minVal = Math.min(...candidates);
    dp[i] = minVal;

    // Update pointers and candidates for all primes that produced minVal
    for (let j = 0; j < primes.length; j++) {
      if (candidates[j] === minVal) {
        // Move pointer forward for this prime
        pointers[j]++;
        // Update candidate: prime × next super ugly number
        candidates[j] = primes[j] * dp[pointers[j]];
      }
    }
  }

  return dp[n - 1];
}

class Solution {
    /**
     * Returns the nth super ugly number.
     * @param n The position of the super ugly number to find (1-indexed)
     * @param primes Array of prime factors allowed
     * @return The nth super ugly number
     */
    public int nthSuperUglyNumber(int n, int[] primes) {
        // dp[i] stores the (i+1)th super ugly number (0-indexed)
        int[] dp = new int[n];
        dp[0] = 1;  // First super ugly number is always 1

        // pointers[i] stores the index in dp that primes[i] should multiply with next
        int[] pointers = new int[primes.length];

        // candidates[i] stores the next candidate value from the stream for primes[i]
        int[] candidates = new int[primes.length];

        // Initialize candidates: each prime multiplied by the first super ugly number (1)
        for (int i = 0; i < primes.length; i++) {
            candidates[i] = primes[i] * dp[0];
        }

        // Generate the remaining n-1 super ugly numbers
        for (int i = 1; i < n; i++) {
            // Find the minimum candidate value
            int minVal = Integer.MAX_VALUE;
            for (int candidate : candidates) {
                minVal = Math.min(minVal, candidate);
            }
            dp[i] = minVal;

            // Update pointers and candidates for all primes that produced minVal
            for (int j = 0; j < primes.length; j++) {
                if (candidates[j] == minVal) {
                    // Move pointer forward for this prime
                    pointers[j]++;
                    // Update candidate: prime × next super ugly number
                    candidates[j] = primes[j] * dp[pointers[j]];
                }
            }
        }

        return dp[n - 1];
    }
}

## Complexity Analysis

**Time Complexity: O(n × k)**

- We need to generate n super ugly numbers
- For each number, we:
  1. Find the minimum among k candidates: O(k)
  2. Update pointers for all primes that produced the minimum: in worst case O(k)
- Total: O(n × k), where k = number of primes

**Space Complexity: O(n + k)**

- `dp` array stores n super ugly numbers: O(n)
- `pointers` array of size k: O(k)
- `candidates` array of size k: O(k)
- Total: O(n + k)

We can optimize the minimum finding to O(log k) using a min-heap, but in practice, for typical LeetCode constraints, the O(n × k) approach is usually sufficient and simpler to implement.

## Common Mistakes

1. **Forgetting that 1 is always the first super ugly number**: Some candidates start their sequence from the smallest prime, but by definition, 1 has no prime factors (or an empty set of prime factors), so it qualifies.

2. **Not handling duplicate candidates correctly**: When multiple primes produce the same minimum value (e.g., 2×7 and 7×2 both equal 14), we need to advance pointers for ALL of them. Otherwise, we'll generate duplicates in our sequence.

3. **Off-by-one errors with array indices**: Remember that `dp[0]` corresponds to the 1st super ugly number, and `dp[n-1]` corresponds to the nth. The loop should run from 1 to n-1 inclusive.

4. **Using integer overflow**: Although the problem guarantees the result fits in 32-bit, intermediate calculations (prime × dp[pointer]) might overflow if we use the wrong data type. In Java, use `long` for candidates if needed, though for this problem's constraints, int is sufficient.

## When You'll See This Pattern

This k-way merge pattern appears in several problems where you need to generate a sequence from multiple sorted sources:

1. **Ugly Number II (LeetCode 264)**: The exact same problem but with fixed primes [2,3,5]. This is a special case of Super Ugly Number.

2. **Find K Pairs with Smallest Sums (LeetCode 373)**: You maintain pointers to merge multiple sorted lists of pair sums.

3. **Merge k Sorted Lists (LeetCode 23)**: The fundamental k-way merge problem that this pattern is based on.

4. **Kth Smallest Element in a Sorted Matrix (LeetCode 378)**: Each row is a sorted list, and you need to find the kth smallest element across all rows.

The pattern is: when you need to generate elements in sorted order from multiple sorted sequences, maintain pointers/indexes for each sequence and repeatedly take the smallest available next element.

## Key Takeaways

1. **Dynamic programming with multiple pointers** is the key technique: Each prime gets its own pointer into the sequence of super ugly numbers, and we always multiply the prime with the super ugly number at its pointer to generate candidates.

2. **Think in terms of merging sorted streams**: Every prime generates a sorted stream of multiples, and finding the next super ugly number is equivalent to taking the smallest available number across all streams.

3. **Watch for duplicates**: When multiple streams produce the same value, advance all corresponding pointers to avoid duplicates in the final sequence.
