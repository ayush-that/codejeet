## How to Solve Subsets II

This problem asks us to generate all possible subsets (the power set) from an array that may contain duplicate elements, while ensuring no duplicate subsets appear in the final result. The challenge lies in efficiently handling duplicates—if we simply generate all subsets like in the classic Subsets problem, we'll produce identical subsets from duplicate numbers, which violates the requirement.

**What makes this tricky:** The presence of duplicates means we can't just generate all 2ⁿ combinations. We need a way to skip over duplicate elements at certain decision points to avoid generating identical subsets.

---

## Visual Walkthrough

Let's trace through a concrete example: `nums = [1, 2, 2]`

If we generate all subsets naively (including treating both 2's as distinct), we'd get:

- `[]`
- `[1]`
- `[2]` (from first 2)
- `[2]` (from second 2) ← DUPLICATE!
- `[1, 2]` (1 with first 2)
- `[1, 2]` (1 with second 2) ← DUPLICATE!
- `[2, 2]` (both 2's together)
- `[1, 2, 2]` (all elements)

The duplicates occur because when we're at a decision point (choose or skip an element), if the current element equals the previous element **and** we didn't include the previous element in our current path, choosing the current element would create a duplicate subset.

**Key insight:** Sort the array first. Then, when backtracking, if `nums[i] == nums[i-1]` and we're at a point where we skipped `nums[i-1]`, we should also skip `nums[i]` to avoid duplicates.

Let's walk through the sorted array `[1, 2, 2]` with proper duplicate handling:

1. Start with empty subset `[]`
2. At index 0 (value 1):
   - Include 1 → `[1]`
   - Skip 1 → stay at `[]`
3. At index 1 (value 2):
   - From `[]`: include 2 → `[2]`
   - From `[]`: skip 2 → stay `[]`
   - From `[1]`: include 2 → `[1, 2]`
   - From `[1]`: skip 2 → stay `[1]`
4. At index 2 (value 2):
   - From `[2]`: include 2 → `[2, 2]` ✓ (allowed because we included the previous 2)
   - From `[2]`: skip 2 → stay `[2]`
   - From `[]`: **skip** including 2 → this avoids duplicate `[2]`
   - From `[]`: skip 2 → stay `[]`
   - From `[1, 2]`: include 2 → `[1, 2, 2]` ✓
   - From `[1, 2]`: skip 2 → stay `[1, 2]`
   - From `[1]`: **skip** including 2 → this avoids duplicate `[1, 2]`
   - From `[1]`: skip 2 → stay `[1]`

Final subsets: `[[], [1], [2], [1, 2], [2, 2], [1, 2, 2]]` (6 unique subsets)

## Brute Force Approach

A naive approach would be to generate all 2ⁿ subsets (where n = length of nums) using bit manipulation or recursion, then use a set to filter out duplicates. While this would technically work, it's inefficient because:

1. We generate many duplicate subsets only to discard them later
2. Using a set to store subsets requires converting lists to tuples (in Python) or similar transformations, adding overhead
3. The time complexity becomes O(2ⁿ × n) for generation plus additional overhead for duplicate checking

The brute force misses the opportunity to avoid generating duplicates in the first place, which is what interviewers look for.

---

## Optimized Approach

The optimal solution uses **backtracking with pruning**. Here's the step-by-step reasoning:

1. **Sort the array first** - This groups duplicates together, making them easy to identify and handle
2. **Use backtracking** - Explore all combinations by making a decision at each index: include the current element or skip it
3. **Prune duplicate branches** - When we encounter a duplicate element (`nums[i] == nums[i-1]`), we should only include it in our current path if we included the previous duplicate. If we skipped the previous duplicate, skipping the current one as well avoids generating duplicate subsets.

**Why this works:** When we have consecutive duplicates `[x, x]` and we skip the first `x`, any subset we generate by including the second `x` would be identical to a subset we could have generated by including the first `x` instead. By skipping the second `x` when we skipped the first, we ensure each unique subset is generated exactly once.

## Optimal Solution

<div class="code-group">

```python
# Time: O(n * 2^n) - We generate 2^n subsets, and copying each subset takes O(n) time
# Space: O(n) - For the recursion stack and current subset (excluding output space)
def subsetsWithDup(nums):
    # Sort the array to group duplicates together
    nums.sort()

    result = []
    current_subset = []

    def backtrack(start_index):
        # Add a copy of the current subset to results
        # This captures the subset at the current decision point
        result.append(current_subset.copy())

        # Explore all possible next elements to add
        for i in range(start_index, len(nums)):
            # Skip duplicates: if current element equals previous AND
            # we're not the first element in this level of recursion
            if i > start_index and nums[i] == nums[i-1]:
                continue

            # Include nums[i] in the current subset
            current_subset.append(nums[i])

            # Recurse to explore subsets that include nums[i]
            # i + 1 ensures we don't reuse the same element
            backtrack(i + 1)

            # Backtrack: remove nums[i] to explore subsets without it
            current_subset.pop()

    # Start backtracking from index 0
    backtrack(0)
    return result
```

```javascript
// Time: O(n * 2^n) - We generate 2^n subsets, and copying each subset takes O(n) time
// Space: O(n) - For the recursion stack and current subset (excluding output space)
function subsetsWithDup(nums) {
  // Sort the array to group duplicates together
  nums.sort((a, b) => a - b);

  const result = [];
  const currentSubset = [];

  function backtrack(startIndex) {
    // Add a copy of the current subset to results
    // This captures the subset at the current decision point
    result.push([...currentSubset]);

    // Explore all possible next elements to add
    for (let i = startIndex; i < nums.length; i++) {
      // Skip duplicates: if current element equals previous AND
      // we're not the first element in this level of recursion
      if (i > startIndex && nums[i] === nums[i - 1]) {
        continue;
      }

      // Include nums[i] in the current subset
      currentSubset.push(nums[i]);

      // Recurse to explore subsets that include nums[i]
      // i + 1 ensures we don't reuse the same element
      backtrack(i + 1);

      // Backtrack: remove nums[i] to explore subsets without it
      currentSubset.pop();
    }
  }

  // Start backtracking from index 0
  backtrack(0);
  return result;
}
```

```java
// Time: O(n * 2^n) - We generate 2^n subsets, and copying each subset takes O(n) time
// Space: O(n) - For the recursion stack and current subset (excluding output space)
import java.util.*;

public class Solution {
    public List<List<Integer>> subsetsWithDup(int[] nums) {
        // Sort the array to group duplicates together
        Arrays.sort(nums);

        List<List<Integer>> result = new ArrayList<>();
        List<Integer> currentSubset = new ArrayList<>();

        backtrack(nums, 0, currentSubset, result);
        return result;
    }

    private void backtrack(int[] nums, int startIndex,
                          List<Integer> currentSubset,
                          List<List<Integer>> result) {
        // Add a copy of the current subset to results
        // This captures the subset at the current decision point
        result.add(new ArrayList<>(currentSubset));

        // Explore all possible next elements to add
        for (int i = startIndex; i < nums.length; i++) {
            // Skip duplicates: if current element equals previous AND
            // we're not the first element in this level of recursion
            if (i > startIndex && nums[i] == nums[i - 1]) {
                continue;
            }

            // Include nums[i] in the current subset
            currentSubset.add(nums[i]);

            // Recurse to explore subsets that include nums[i]
            // i + 1 ensures we don't reuse the same element
            backtrack(nums, i + 1, currentSubset, result);

            // Backtrack: remove nums[i] to explore subsets without it
            currentSubset.remove(currentSubset.size() - 1);
        }
    }
}
```

## Complexity Analysis

**Time Complexity:** O(n × 2ⁿ)

- Sorting takes O(n log n), which is dominated by the subset generation
- We generate at most 2ⁿ subsets (in the worst case with no duplicates)
- Creating a copy of each subset to add to results takes O(n) time
- Total: O(n log n + n × 2ⁿ) = O(n × 2ⁿ)

**Space Complexity:** O(n)

- The recursion stack depth is at most n
- The `currentSubset` list grows to at most size n
- **Note:** This doesn't include the space for the output, which is O(n × 2ⁿ) to store all subsets

## Common Mistakes

1. **Forgetting to sort the array first** - This is crucial for the duplicate skipping logic to work. Without sorting, duplicates won't be adjacent, so `nums[i] == nums[i-1]` won't catch all duplicates.

2. **Incorrect duplicate skipping condition** - Using `if i > 0 and nums[i] == nums[i-1]` instead of `if i > startIndex and nums[i] == nums[i-1]`. The latter is correct because we want to allow duplicates when we're starting a new branch (i == startIndex) but skip them when we're continuing in the same branch.

3. **Not making a copy of the current subset** - Adding `currentSubset` directly to results means all subsets in results will reference the same list object, which gets modified during backtracking. Always add a copy: `list(currentSubset)` in Python, `[...currentSubset]` in JavaScript, or `new ArrayList<>(currentSubset)` in Java.

4. **Using bit manipulation without duplicate handling** - Some candidates try to adapt the bitmask approach from Subsets I, but this becomes complex with duplicates. Backtracking with pruning is cleaner for this variant.

## When You'll See This Pattern

This "backtracking with pruning to handle duplicates" pattern appears in several combinatorial problems:

1. **Combination Sum II** (LeetCode 40) - Find all unique combinations where candidate numbers sum to target, with each number used once. Requires sorting and skipping duplicates just like Subsets II.

2. **Permutations II** (LeetCode 47) - Generate all unique permutations of a collection that might contain duplicates. Uses a similar "skip if duplicate and previous wasn't used" logic.

3. **Palindrome Partitioning II** (LeetCode 131) - While not about duplicates, it uses similar backtracking to explore all partitioning possibilities.

The core pattern is: when you need to generate combinations/permutations/subsets from elements that may have duplicates, sort first, then use backtracking with checks to skip over duplicate elements at appropriate decision points.

---

## Key Takeaways

1. **Sorting enables efficient duplicate handling** - By grouping duplicates together, we can easily identify when to skip elements to avoid generating duplicate subsets/combinations.

2. **The duplicate skipping condition is subtle but crucial** - We skip `nums[i]` if it equals `nums[i-1]` AND `i > startIndex`. This allows the first occurrence in a new recursive branch but skips subsequent duplicates in the same branch.

3. **Backtracking is often cleaner than bit manipulation for problems with constraints** - While bit manipulation works for the basic Subsets problem, backtracking with pruning handles constraints like duplicates more elegantly.
